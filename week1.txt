1. 앞으로 git 프로젝트로 만들고 싶은 것

만들고 싶었던 프로젝트를 git에 정리 해보고 싶다.


2. 1주차 주요 개념 키워드 적어보기

- 버전 관리	: 작업한 내용을 commit 하면서 내용을 요약해 놓으면, commit 하나하나가 하나의 버전이 된다.
- git 초기화 (initialize, init)	: 로컬에서 진행하던 프로젝트를 git과 연결하는 작업.
- add / staging	: commit 할지말지 정할 때 commit을 하고자 하는 파일을 정하는 것을 add라고 하고, 그 올라가는 파일들을 stage에 올라갔다고 한다.
- commit (커밋)	: 작업 내용의 중간 저장 느낌.
- commit 내역(history)	: commit의 시간순의 리스트. commit의 요약이 잘 되어있으면 아주 유용해진다.
- push	: 로컬 repo의 commit을 원격 repo에.
- pull	: 원격 repo의 commit을 로컬 repo에.
- clone	: 원격 repo를 컴퓨터로 복사.
- tracking	: 로컬과 원격을 연결하는 것을 tracking 이라고 한다.
